# あなたがやること チェックリスト

> Claude Codeが実装できない・あなたがやった方が早い作業を全て網羅したリストです。
> 各Phaseの開始前にこのリストを見て、必要な作業を済ませてから Claude Code を呼んでください。

---

## 全体の流れと「あなたがやること」の分布

```
Phase 0  ← ★ここが一番多い・一番重要
Phase 1  ← Chromeを起動してログインするだけ（毎回）
Phase 2  ← Phase 1 と同じ
Phase 3  ← Phase 0 と同様の DOM 調査が追加で必要
Phase 4  ← Dry-run の Y/n を判断するだけ
```

### □ P0-1: Node.js v20 以上の確認

**所要時間:** 確認1分 / インストール必要なら15分

```bash
# ターミナルで実行
node --version
# → "v20.xx.x" 以上が表示されれば OK
# → v18 以下または "command not found" の場合 → nodejs.org から LTS 版をインストール

npm --version
# → 何かバージョンが表示されれば OK
```

**失敗すると:** npm が使えないため開発作業が全てブロックされる。

---

### □ P0-2: Google Chrome（通常版）の確認

**所要時間:** 確認1分 / インストール必要なら15分

```
Finder → アプリケーション → "Google Chrome.app" が存在するか確認

存在しない場合: google.com/chrome からダウンロード・インストール

⚠ 注意: "Google Chrome Canary.app" は不可。通常版が必要。
```

**失敗すると:** CDP 接続ができず、ツール全体が動かない。

**所要時間:** 確認1分 / インストール必要なら15分

## ■ Phase 0-A: DOM 調査スパイク（最重要・最も時間がかかる）

> ここが全体で**一番重要な人間作業**です。

### □ 0A-1: Chrome をデバッグポート付きで起動する

**所要時間:** 5分（毎回の実行前にも同じ手順が必要）

```bash
# ステップ1: 開いているChromeを全て終了
# Dock の Chrome アイコンを右クリック → 「終了」（または Cmd + Q）

# ステップ2: ターミナルで以下を1行で実行
```

### □ 0A-3: シナリオ一覧ページのセレクタ調査

**所要時間:** 20〜30分

**DevTools の使い方（基本手順）:**
```
1. シナリオ管理ページを開く
2. F12 キー（または右クリック → 「検証」）で DevTools を開く
3. 「Elements」タブをクリック
4. 調査したい要素（テーブル・ボタン等）にマウスを乗せる
5. Elements タブで青くハイライトされた要素を右クリック
6. Copy → Copy selector をクリック
7. メモ帳に貼り付け
```

**調査する要素（全部で5箇所）:**

| 要素 | 調査方法 | セレクタメモ欄 |
|------|---------|--------------|
| シナリオ一覧テーブル全体 | テーブルの外枠を選択 | |
| テーブルの1行（シナリオ1件分） | 1つの行を選択 | |
| 行内のシナリオ名（リンク） | シナリオ名テキストを選択 | |
| 行内のステータス表示 | "配信中"等のテキストを選択 | |
| 複製ボタン（あれば） | 「複製」ボタンを選択 | |

**コピーしたセレクタの判断:**
- `#scenario-list-table` や `[data-test="scenarios"]` のような形式 → 安定している（使える）
- `#main > div:nth-child(3) > table` のような数字の多い形式 → 不安定（Claude Code に相談）

調査後: メモしたセレクタを Claude Code に共有する

## ■ Phase 4: 書き込み操作時の判断（Dry-run確認）

---

### □ PH4-1: Dry-run プロンプトでの Y/n 判断

**毎回発生します。**

```
スクリプト実行中に以下のプロンプトが表示されたら:

  [DRY RUN] 以下の操作を実行します:
    ① シナリオ「初回フォロー」を複製 → 「初回フォロー_コピー」

  依存関係への影響:
    ⚠ タグ「新規」はシナリオB・Cでも参照されています
    ✓ リッチメニューへの影響なし

  実行しますか？ (Y/n):

判断基準:
  ① 操作内容が自分の指示した通りか確認
  ② ⚠ 警告がある場合 → 影響を受けるシナリオ・タグが何か確認し、問題ないか判断
  ③ 問題なければ Y + Enter
  ④ 迷ったら必ず n + Enter で中止（後から再実行できる）
```

## ■ セッション切れ・エラー発生時の対処

---

### □ エラー対処: セッション切れが発生したとき

```
症状: "Navigation to login page detected" または "Session expired" が表示される

対処:
1. Ctrl + C でスクリプトを停止
2. Chrome を確認してログインし直す
3. スクリプトを最初から再実行
```

---

### □ エラー対処: セレクタエラーが突然発生したとき

```
症状: "Element not found: scenarios.listTable" 等が表示される

対処:
1. Chrome を手動で開いてそのページにアクセス
2. DevTools で該当要素が変わっていないか確認
3. 変わっていた場合 → 作業 0A-3〜0A-5 と同じ手順で再調査
4. 新しいセレクタを Claude Code に共有
```

## ■ 運用・保守

---

### □ 定期: Phase 完了時のコミット

```
各 Phase の品質ゲートを通過したら Claude Code に依頼:
「Phase X が完了したのでコミットしてください」

```

---

### □ 定期: selectors.json のメンテナンス（月1回または壊れたとき）

```
Lステップが画面更新すると selectors.json のセレクタが壊れる可能性がある。
月1回、または突然スクリプトがエラーを出したときに:

1. Chrome デバッグ起動 → Lステップにログイン
2. npm run scan:scenarios（または他のコマンド）を実行
3. エラーが出たページのセレクタを再調査（0A-3 の手順）
4. 新しいセレクタを Claude Code に共有 → selectors.json を更新
```
