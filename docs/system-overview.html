データ取得日: 2026-03-21 | シナリオ18件 / タグ111件 / テンプレート58種 / 友だち情報43件

30秒でわかるこのシステム

このLINEアカウントは、矯正歯科に興味がある人が友だち登録してから初診予約するまでを、
自動メッセージ配信で支援するシステムです。
患者さんの「悩み」「興味度」「段階」に応じて、最適なメッセージが自動で届きます。

日常のたとえで理解する「Lステップの4つの部品」

📋
シナリオ = レシピ
「1日目にAを送る、3日後にBを送る」という手順書。料理のレシピのように順番が決まっている。
🏷️
タグ = 付箋（ふせん）
患者さんに貼る分類ラベル。「出っ歯で悩んでいる」「福岡の人」など。この付箋を見て、どのレシピを使うか決める。
✉️
テンプレート = 手紙の文面
実際に送るメッセージの中身。同じ文面を複数のレシピで使い回せる（共有テンプレート）。
📇
友だち情報 = カルテ
患者さんごとの個別データ。予約URL・登録日・ウェビナー参加日など。このデータで配信タイミングを計算する。

1. 患者さんの体験フロー（全体の流れ）

「患者さんの目線」で見た、LINE登録から予約・契約までの流れです。各ステップで何が起きているかを追えます。
1
LINE友だち登録
患者さんがInstagram広告・LP・YouTube・QRコードなどからLINE公式アカウントを友だち追加する。
どこから来たかを自動で記録。
🏷️ 流入経路タグが自動で付く
2
登録時アンケート
友だち追加した直後にアンケートが表示される。
「お悩みの症例は?」「希望の医院は?」「相談の緊急度は?」の3問に回答。
回答内容に応じてタグが自動付与され、以降の配信内容が決まる。
🏷️ 症例タグ（例: 叢生、出っ歯） 🏷️ 医院タグ（例: 福岡、札幌） 🏷️ 意欲タグ（例: 今すぐ相談したい）
3
カウンセリング誘導（1時間後〜）
登録の約1時間後から自動配信が始まる。
「矯正治療についてもっと詳しく知りませんか?」とウェビナー（説明動画）への参加を誘導。
患者さんのフェーズ移行日時がカルテに記録される。
📇 フェーズ移行日時をカルテに記録 ✉️ カウンセリング誘導テンプレート使用
4
オートウェビナー（録画動画視聴）
矯正治療の流れ・費用・症例紹介をまとめた録画動画を案内。
視聴の進捗（何%まで見たか）が自動で記録される。
視聴率が高い人 = 関心度が高い、と判定してフォロー強度を変える。
🏷️ 視聴率タグ（0%/5%/10%/.../100%） 🏷️ AW申込済み / AW参加 タグ 📇 ウェビナー参加URL・申込日を記録
5
症例動画ステップ配信（数日おき × 4本）
アンケートで選んだ症例に合ったBefore/After動画が数日おきに届く。
例: 「叢生」を選んだ人 → 叢生#1 → 叢生#2 → 叢生#3 → 叢生#4 の4本。
9症例ごとに別のシナリオ（レシピ）が用意されている。
✉️ 症例別テンプレート（例: 叢生#1〜#4）
6
予約誘導CTA（カウンセリング予約へ）
症例動画の最後に「カウンセリングを予約しませんか?」のメッセージが届く。
この予約誘導メッセージは10箇所で共有されている最重要テンプレート。
予約URLをタップするとクリック数が記録され、予約完了でゴール達成。
✉️ 共有テンプレート「予約誘導」（10箇所共有） 🏷️ 予約URLクリック済み 🎯 初診-予約済み（ゴール!） 📇 クリック回数+1
7
離脱防止フォロー（初診後〜契約まで）
初診後に「精密検査」や「契約」を迷っている患者さんへ、
当日/1日後/3日後/7日後/14日後 にフォローメッセージを自動配信。
30日間反応がなければ最後のリマインドを1回送信。
来院時のQRスキャンで進捗ステータスが更新される。
✉️ 精密/契約検討テンプレート 🏷️ QRスキャン-予約ステータスタグ 📇 配信日時・フェーズ移行日時を管理

具体例: 「叢生（歯並びガタガタ）」で悩むAさんの場合

  1. Instagram広告からLINE登録 → 🏷️「流入経路-広告」タグが付く
  2. アンケートで「叢生」「福岡」「今すぐ相談したい」を選択 → 🏷️ 3つのタグが付く
  3. 1時間後「カウンセリング誘導」シナリオが自動で配信開始 📇 フェーズ日時を記録
  4. ウェビナー（矯正の説明動画）を案内 → 視聴すると 🏷️「滞在率50%」等のタグが付く
  5. 「叢生」の症例動画を4本、数日おきに自動配信 ✉️ 叢生#1 → #2 → #3 → #4
  6. 最後に予約誘導メッセージ ✉️「ステップ配信後-予約誘導」（共有テンプレート）
  7. 予約URLをタップ → 🏷️「福岡MA-予約URL-クリック済み」タグ → 📇 クリック回数+1
ポイント: タグが全体をつなぐ「血管」
各ステップでタグが付与され、そのタグを条件に次のシナリオが発動します。タグを変更すると下流のシナリオ全体に影響が波及するため、変更時は依存関係の確認が必須です。

4. 依存関係マップ — 何が何とつながっているか

「シナリオAがテンプレートBを使っている」という関係を全104件可視化しています。
同じテンプレートを複数シナリオが使っている = 共有テンプレート。変更すると全シナリオに影響します。

「共有テンプレート」とは何か？

たとえ話: 社内で1枚の「契約書ひな形」を10部署が使っているとします。
ひな形を修正すると10部署全ての契約書が変わります。これが「共有テンプレート」のリスクです。
逆に、1箇所直せば全部直る効率性でもあります。
