# Lステップ構造ガイド

このスキルは、Lステップ（矯正歯科LINE公式アカウント）の構造を理解し、変更時の影響をチェックするためのガイドです。

## 起動時に必ず実行すること

1. `.claude/skills/lstep-guide/knowledge.md` を Read してナレッジベースをロードする
2. ユーザーの質問タイプを判定する（下記分類）
3. 該当する対応パターンに従って回答する

## 質問タイプと対応パターン

### タイプA: 構造の質問（「今どうなってる?」系）

**トリガー例**: 「タグ一覧を見せて」「シナリオはいくつある?」「依存関係を教えて」「全体の構造は?」

**対応**:
1. knowledge.md から該当セクションを参照して回答
2. 必要に応じて `data/` 内の最新JSONファイルも Read して補足
3. 件数・名前・関係性を表形式で整理して提示
4. 全体像が必要なら `docs/system-overview.html` の存在を案内（`open` で開ける）

### タイプB: 新規作成ガイド（「○○を作りたい」系）

**トリガー例**: 「新しいシナリオを追加したい」「タグを新しく作りたい」「テンプレートを追加したい」

**対応**:
1. 何を作りたいかヒアリング（名前・目的・対象者）
2. knowledge.md の依存関係セクションを参照し、以下をチェック:
   - 既存のタグ・テンプレート・シナリオとの命名衝突がないか
   - 再利用できる既存テンプレートがあるか
   - 既存の共有テンプレートを使う場合の影響範囲
3. チェック結果を踏まえた**作成計画書**を提示:

```
## 作成計画: {作成するものの名前}

### 概要
- 種類: シナリオ / タグ / テンプレート / 友だち情報
- 目的: {なぜ作るか}

### 依存関係チェック
- [ ] 命名衝突: なし / あり（{衝突先}）
- [ ] 再利用可能な既存リソース: {リスト}
- [ ] 影響を受ける既存シナリオ: {リスト}

### 作成手順
1. {手順}
2. {手順}
3. {手順}

### 作成後の確認
- [ ] {確認項目}
```

4. ユーザーが承認したら具体的な手順に進む

### タイプC: 変更・編集ガイド（「○○を変えたい」系）

**トリガー例**: 「このテンプレートを編集したい」「タグ名を変えたい」「シナリオを止めたい」

**対応**:
1. 何を変えたいかを特定
2. knowledge.md の「全エッジ一覧」「変更時のチェックリスト」を参照し、**影響範囲を洗い出す**
3. 影響範囲を以下の形式で警告:

```
## 変更影響レポート: {変更内容}

### 影響範囲
| 影響先 | 種類 | 影響内容 | 重要度 |
|--------|------|---------|--------|
| {名前} | シナリオ/テンプレート | {何が変わるか} | 高/中/低 |

### リスク
- {リスク説明}

### 推奨アプローチ
- A: {安全な方法}
- B: {効率的な方法}
→ 推奨: {推奨案と理由}
```

4. 特に以下の場合は**赤字レベルの警告**を出す:
   - 「ステップ配信後-予約誘導-カウン...」テンプレートの変更（10箇所に影響）
   - 友だち情報フィールド名の変更（配信タイミングが壊れる）
   - 症例テンプレート#1-#4の変更（個別シナリオ + AW誘導の2箇所に影響）

### タイプD: 分析・比較（「どれが重要?」「効果は?」系）

**トリガー例**: 「一番使われてるテンプレートは?」「タグの稼働率は?」「改善できるところは?」

**対応**:
1. knowledge.md のデータから集計・分析
2. 必要に応じて `data/` の最新JSONを Read して正確な数値を取得
3. 分析結果を表形式 + インサイト付きで提示
4. planner.ts の関数が使える場合は `npx ts-node` で実行して最新分析を取得

## データ鮮度チェック

回答前にデータの新しさを確認する:
1. `data/` 内の最新ファイルのタイムスタンプを確認（ファイル名に日時が入っている）
2. 24時間以内 → そのまま使用
3. 24時間超 → 「データは{日付}時点のものです。最新に更新するにはChrome接続+スクレイプが必要です」と注記

## 注意事項

- このナレッジベースはスクレイプ時点のスナップショット。Lステップ管理画面で直接変更された場合は反映されていない可能性がある
- タグの「友だち数」は変動する値。傾向の参考にはなるが正確な現在値ではない
- シナリオのステータスは全て「下書き」だった（2026-03-21時点）

## Quick Reference

### 数字で見る現在の構成
```
シナリオ 18本 | タグ 111件 | テンプレート 58種 | 友だち情報 43件 | 依存 104エッジ
```

### シナリオ分類（18本）
```
A. 症例動画ステップ配信 ×9本  … 症例別BA動画4本+予約誘導
B. オートウェビナー誘導  ×2本  … AW視聴→症例配信→予約
C. 離脱防止ステップ     ×4本  … 精密/契約検討フォロー+30日リマインド
D. その他               ×3本  … アンケート離脱防止/YouTube紹介/未実装
```

### タグ分類（111件）
```
流入経路              19件  … 流入経路分析で自動付与
LINE登録時アンケート   16件  … 友だち追加時設定で自動付与
CV・予約完了           6件  … 予約完了アクションで付与
予約URLクリック         2件  … テンプレートURLタップ時
AW視聴率(初診誘導)    14件  … ウェビナー連携で自動付与
AW視聴率(初診後)       9件  … 同上
QRスキャン            20件  … 来院時QRスキャン
YouTube                2件  … リンクタップ時
院長矯正相談           7件  … 専用フロー内
口コミ依頼             7件  … 専用フロー内
その他                 4件  … 性別・送信済み等
```

### 危険な変更 TOP3（要確認）
```
1. 「ステップ配信後-予約誘導-カウン...」 → 10箇所で使用。変更で全症例+AW誘導に波及
2. 症例テンプレート#1-#4                → 各2箇所（個別シナリオ+AW誘導48ステップ）
3. 友だち情報フィールド名               → シナリオ内変数参照が壊れる。名前変更禁止
```

### よく使うコマンド
```
# 全体構成マップをブラウザで開く
open docs/system-overview.html

# planner.tsで依存分析（Chrome接続不要）
npx ts-node src/test-planner.ts

# データ再取得（Chrome接続+Lステップログイン必要）
npx ts-node src/test-connection.ts
```

```

src/
  planner.ts      … 依存分析エンジン（analyze/searchNodes/checkDataFreshness）

  system-overview.html  … 人間向け全体構成マップ（ブラウザ閲覧用）
```
